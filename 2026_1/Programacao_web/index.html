2) Estrutura básica de um documento HTML

<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Título da Página</title>
</head>
<body>
  Conteúdo visível ao usuário
</body>
</html>
  • head: metadados, título, CSS e configurações.
  • body: tudo que aparece na página.
  • title: nome que aparece na aba do navegador.

4) Links, imagens e listas

Links

<a href="pagina.html">Abrir</a>

Use target="_blank" para abrir em nova aba.

Imagem

<img src="foto.jpg" alt="Descrição">

O atributo alt ajuda acessibilidade e descrição da imagem.

Listas

<ol>
  <li>Item</li>
</ol>

ol é ordenada e ul é não ordenada.

5) Tabelas

As tabelas organizam dados em linhas e colunas.

<table>
  <tr>
    <th>Nome</th>
    <th>Idade</th>
  </tr>
  <tr>
    <td>Ana</td>
    <td>20</td>
  </tr>
</table>
  • tr: linha
  • th: célula de cabeçalho
  • td: célula comum
  • colspan: ocupa várias colunas
  • rowspan: ocupa várias linhas

7) GET x POST

Método Quando usar Observação
GET Consulta, pesquisa e filtros Dados aparecem na URL
POST Cadastro, login, envio de dados Dados vão no corpo da requisição

8) CSS: formas de uso

Inline

Direto na tag com o atributo style.

Interno

Dentro da tag style no head.

Externo

Arquivo separado .css, que é a melhor prática na maioria dos casos.

p { color: blue; }
.destaque { color: green; }
#titulo { color: red; }

9) Seletores e especificidade

  • Elemento: seleciona todas as tags daquele tipo. Ex.: p
  • Classe: pode ser reutilizada em vários elementos. Ex.: .caixa
  • ID: deve ser único na página. Ex.: #menu
Prioridade comum: ID > classe > elemento. O uso de !important deve ser raro.

10) Box model e layout

Todo elemento HTML pode ser pensado como uma caixa.

  • content: conteúdo
  • padding: espaço interno
  • border: borda
  • margin: espaço externo
.caixa {
  margin: 10px;
  padding: 20px;
  border: 1px solid #000;
}

11) Responsividade e @media

Responsividade é fazer o site se adaptar ao tamanho da tela. Esse assunto tem grande chance de aparecer porque já conversa com o conteúdo de CSS.

@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
  .grid {
    grid-template-columns: 1fr;
  }
}
  • max-width: 768px significa: aplicar as regras em telas menores.
  • É muito usado para celular.
  • Ajuda a evitar site quebrado em telas pequenas.

12) Erros comuns que a prova pode explorar

  • Usar tags inválidas, como h999.
  • Colocar heading dentro de p.
  • Escrever atributo errado, como nome no lugar de name.
  • Duplicar o mesmo id em vários elementos.
  • Escrever meta viewport incorretamente.
  • Esquecer label em campos de formulário.

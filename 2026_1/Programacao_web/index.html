4) HTML e HTML5 - explicação de prova

4.1 Estrutura mínima correta

<!DOCTYPE html>
<html lang="pt-BR">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Minha Página</title>
</head>
<body>
  Conteúdo da página
</body>
</html>

4.2 Tags essenciais e função

Tag Função
h1 a h6Títulos e hierarquia do conteúdo.
pParágrafo de texto.
aLink interno ou externo.
imgExibir imagem (sempre use alt).
ol / ul / liListas ordenadas e não ordenadas.
table / tr / th / tdDados em formato tabular.
formColetar dados do usuário.

4.3 Semântica no HTML5

Em HTML5, além de funcionar, a página precisa fazer sentido estrutural. Tags semânticas melhoram leitura, SEO e acessibilidade.

header

Cabeçalho da página/seção.

nav

Menu de navegação.

main

Conteúdo principal.

section

Bloco temático de conteúdo.

article

Conteúdo independente.

footer

Rodapé da página/seção.

5) Formulários e controles (ponto forte da prova)

5.1 Estrutura padrão

<form action="resposta.php" method="post">
  <label for="nome">Nome</label>
  <input id="nome" name="nome" type="text" required />

  <label for="email">E-mail</label>
  <input id="email" name="email" type="email" />

  <button type="submit">Enviar</button>
</form>

5.2 GET x POST

Método Uso comum Como vai o dado
GET Busca, filtro, consulta Parâmetros aparecem na URL
POST Cadastro, login, envio sensível Dados vão no corpo da requisição

5.3 Inputs que você precisa lembrar

Tipos clássicos

  • text, password, number
  • radio, checkbox
  • submit, reset, button
  • textarea, select, option
  • fieldset, legend, datalist

Tipos HTML5 (do ZIP)

  • email, url, tel
  • date, month, week, time
  • datetime-local
  • color, range, search

Atributos importantes

  • name: nome da variável enviada
  • id: identificação no HTML/CSS
  • placeholder: dica visual
  • required: obrigatório
  • readonly x disabled: readonly envia, disabled não envia
  • min, max, maxlength, step

6) CSS completo para revisão

6.1 Formas de aplicar CSS

  • Inline: na própria tag com style="...".
  • Interno: bloco <style> no <head>.
  • Externo: arquivo .css separado com <link> (mais usado e organizado).

6.2 Seletores e prioridade

p { color: blue; }        /* seletor por elemento */
.destaque { color: green; } /* seletor por classe */
#titulo { color: red; }      /* seletor por ID */
Regra importante: ID > Classe > Elemento. O !important ganha de quase tudo, mas deve ser usado com cuidado.

6.3 Propriedades que mais aparecem

Categoria Propriedades
Texto color, font-family, font-size, font-weight, text-align
Caixa width, height, margin, padding, border, border-radius
Fundo background-color, background-image, background-repeat
Layout display, position, flex, grid

6.4 Box Model (sempre cai)

.caixa {
  margin: 10px;   /* espaço externo */
  border: 1px solid #111;
  padding: 20px;  /* espaço interno */
}

7) Design responsivo

Responsivo é adaptar o conteúdo para diferentes telas, especialmente celular. Você já usou isso no próprio index atual com grid e media query.

@media (max-width: 768px) {
  .grid {
    grid-template-columns: 1fr;
  }

  body {
    font-size: 15px;
  }
}
  • meta viewport é obrigatório para layout mobile correto.
  • max-width em imagens evita estourar a tela.
  • Use medidas relativas quando possível: %, rem, vw.

8) Erros clássicos que professor costuma cobrar

  • Escrever atributo errado: usar nome em vez de name no input.
  • Meta viewport com digitação incorreta, por exemplo devide-width.
  • Duplicar o mesmo id em elementos diferentes.
  • Colocar heading (h1, h2) dentro de p.
  • Usar disabled achando que o valor será enviado no form.
  • Esquecer label, prejudicando acessibilidade e foco.

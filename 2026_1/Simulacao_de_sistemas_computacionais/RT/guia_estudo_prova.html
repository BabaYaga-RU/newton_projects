0) Comecando do Zero (sem base previa)

Se as formulas parecem codigos aleatorios, use este roteiro sempre:

  1. Leia a pergunta em portugues simples: o que estao pedindo para descobrir?
  2. Liste os dados: escreva os numeros que voce recebeu.
  3. Escolha a ferramenta: media (centro), desvio (espalhamento), fila (espera), etc.
  4. Calcule com calma: uma linha por vez, sem pular etapa.
  5. Interprete: termine com "isso indica que...".
Regra pratica: na prova, conta sem interpretacao perde ponto. Sempre escreva o significado final do numero.

Dicionario rapido de simbolos

  • n: quantidade de valores da amostra.
  • xi: cada valor individual da lista.
  • x_barra: media da amostra (valor central).
  • s^2: variancia amostral (espalhamento ao quadrado).
  • s: desvio padrao (espalhamento na unidade original).
  • CV: variacao relativa em porcentagem.

Como estudar sem travar

  1. Comece por 1 exercicio facil de media/desvio.
  2. Repita o mesmo tipo trocando os numeros.
  3. So depois avance para IC e valor-p.
  4. No final, faca 1 questao de fila (M/M/1).
Erro comum: tentar decorar todas as formulas de uma vez. Entenda primeiro o que cada medida responde.

1) Simulacao de Sistemas Computacionais (Teoria Completa)

Conceito: simulacao e a criacao de um modelo computacional de um sistema real para experimentar cenarios sem afetar o sistema real.

1.1 Conceitos basicos

Termo Definicao Exemplo em computacao
Sistema Conjunto de entidades que interagem para um objetivo. Servidor web atendendo requisicoes.
Modelo Representacao simplificada do sistema real. Fila de requisicoes com taxa media de chegada.
Evento Ocorrencia que muda estado do sistema. Chegada de pacote, fim de atendimento.
Variavel de estado Informacao que descreve o sistema naquele instante. Numero de clientes na fila.

1.2 Etapas da modelagem

  1. Definir problema com pergunta mensuravel.
  2. Selecionar variaveis de entrada e saida.
  3. Escolher nivel de abstracao (nem simples demais, nem complexo demais).
  4. Construir modelo (analitico ou simulacao).
  5. Validar com dados reais.
  6. Rodar cenarios e comparar indicadores.
  7. Concluir e recomendar decisoes.

1.3 Analitico x Simulacao

Aspecto Modelo analitico Simulacao
Complexidade aceita Baixa a media Media a alta
Resultado Fechado/exato (quando possivel) Estimativa estatistica
Tempo de preparacao Alto para formular Alto para modelar e calibrar
Quando usar Sistema com hipoteses fortes Sistema realista com variabilidade

2) Estatistica Aplicada (Teoria + Interpretacao)

2.1 Estatistica descritiva

Serve para responder duas perguntas basicas: qual e o centro dos dados? e o quanto os dados variam?

2.1.1 Traduzindo as formulas para linguagem simples

Media amostral (x_barra): Some todos os valores e divida pela quantidade de valores. x_barra = (x1 + x2 + ... + xn) / n
Variancia amostral (s^2): Veja o quanto cada valor se afasta da media, eleve ao quadrado, some tudo e divida por (n - 1). s^2 = soma((xi - x_barra)^2) / (n - 1)
Desvio padrao (s): E a raiz da variancia. Mostra a "distancia tipica" dos valores ate a media. s = raiz(s^2)
Coeficiente de variacao (CV): Compara o desvio com a media em porcentagem. CV = (s / x_barra) * 100%

2.1.2 Exemplo do zero (bem guiado)

Dados: [10, 12, 8, 10, 10]

Passo 1: media x_barra = (10 + 12 + 8 + 10 + 10) / 5 = 50 / 5 = 10 Passo 2: variancia Desvios para a media: (10-10)=0, (12-10)=2, (8-10)=-2, (10-10)=0, (10-10)=0 Quadrados: 0, 4, 4, 0, 0 Soma dos quadrados = 8 s^2 = 8 / (5-1) = 8/4 = 2 Passo 3: desvio padrao s = raiz(2) = 1,41 Passo 4: coeficiente de variacao CV = (1,41 / 10) * 100 = 14,1% Interpretacao final: Os dados giram em torno de 10, com variacao moderada (CV de 14,1%).
Cuidado de prova: media e sensivel a outliers; mediana e mais robusta. Se aparecer valor muito distante, comente isso na interpretacao.

2.3 Estatistica inferencial

  • Populacao: conjunto total.
  • Amostra: subconjunto usado para inferir sobre a populacao.
  • Intervalo de confianca: faixa plausivel para parametro populacional.
  • Teste de hipotese: compara evidencias da amostra com H0.
Regra comum no teste t: se p < 0,05, rejeita H0 (ha evidencia de diferenca).
Interpretacao correta de valor-p

Valor-p nao e a probabilidade de H0 ser verdadeira. E a probabilidade de observar resultado tao extremo quanto o da amostra assumindo H0 verdadeira.

3) Teoria das Filas (Completo para Prova)

3.1 Elementos do sistema de fila

  • Processo de chegadas (taxa lambda).
  • Processo de servico (taxa mu por servidor).
  • Numero de servidores (k).
  • Disciplina (FIFO, LIFO, prioridade).
  • Capacidade total e populacao.

3.2 Notacao de Kendall

A/B/c/K/m/Z
  • A: distribuicao entre chegadas (M, D, G, Erlang-k).
  • B: distribuicao do tempo de servico.
  • c: numero de servidores.
  • K: capacidade total do sistema.
  • m: tamanho da populacao.
  • Z: disciplina da fila.

3.3 Formulas essenciais M/M/1

rho = lambda / mu (condicao de estabilidade: rho < 1)
L = rho / (1 - rho)
Lq = rho^2 / (1 - rho)
W = 1 / (mu - lambda)
Wq = lambda / (mu * (mu - lambda))
Lei de Little: L = lambda * W e Lq = lambda * Wq

3.4 M/M/k (multiplos servidores)

Utilizacao por servidor: rho = lambda / (k * mu)

Estavel: rho < 1 | Instavel: rho >= 1

Em prova, mesmo sem decorar Erlang-C completo, normalmente voce precisa interpretar efeito de aumentar k: queda de fila/espera e menor risco de colapso.

4) Exercicios Resolvidos (Passo a Passo)

Exercicio 1 - Estatistica descritiva

Dados (tempo de resposta em ms): [100, 105, 98, 102, 101]

Passo 1: media x_barra = (100 + 105 + 98 + 102 + 101) / 5 x_barra = 506 / 5 = 101,2 ms Passo 2: variancia amostral Desvios: -1,2; 3,8; -3,2; 0,8; -0,2 Quadrados: 1,44; 14,44; 10,24; 0,64; 0,04 Soma = 26,80 s^2 = 26,80 / (5 - 1) = 6,70 Passo 3: desvio padrao s = raiz(6,70) = 2,59 ms Passo 4: coeficiente de variacao CV = (2,59 / 101,2) * 100 = 2,56% Conclusao: Baixa variabilidade. Sistema estavel.

Exercicio 2 - Comparacao de amostras (estabilidade)

Amostra A: [100, 105, 98, 102, 101] | Amostra B: [80, 120, 90, 110, 100]

Resumo da dispersao: Amostra A: media ~101,2 e CV ~2,56% Amostra B: media =100, desvio ~15,81 e CV ~15,81% Conclusao: Amostra A e mais estavel. Amostra B apresenta comportamento mais imprevisivel (maior dispersao).

Exercicio 3 - Intervalo de confianca de 95% (media)

Dados de altura (cm): [168, 170, 172, 169, 171, 173, 167, 170, 169, 172]

n = 10 media = 170,1 desvio amostral ~ 1,91 erro padrao = 1,91 / raiz(10) = 0,60 Usando t(0,975; gl=9) ~ 2,262: Margem = 2,262 * 0,60 = 1,36 IC 95% = 170,1 +- 1,36 IC 95% = [168,74 ; 171,46] Interpretacao: Com 95% de confianca, a media populacional esta nesse intervalo.

Exercicio 4 - Teste de hipotese (interpretacao de valor-p)

Suponha teste t entre metodo tradicional e novo metodo retornando p = 0,058 com alfa = 0,05.

Regra: Se p < 0,05 -> rejeita H0 Se p >= 0,05 -> nao rejeita H0 Aplicando: 0,058 > 0,05 Logo: nao rejeitamos H0. Frase correta de prova: "Nao houve evidencia estatistica suficiente para afirmar diferenca entre as medias ao nivel de 5%."

Exercicio 5 - M/M/1 completo

Dado: lambda = 8 clientes/hora, mu = 10 clientes/hora.

Passo 1: utilizacao rho = lambda/mu = 8/10 = 0,8 (estavel, pois < 1) Passo 2: numero medio no sistema L = rho/(1-rho) = 0,8/0,2 = 4 clientes Passo 3: numero medio na fila Lq = rho^2/(1-rho) = 0,64/0,2 = 3,2 clientes Passo 4: tempo medio no sistema W = 1/(mu-lambda) = 1/(10-8) = 0,5 hora = 30 min Passo 5: tempo medio na fila Wq = lambda/(mu*(mu-lambda)) = 8/(10*2) = 0,4 hora = 24 min Cheque com Little: L = lambda*W = 8*0,5 = 4 (ok) Lq = lambda*Wq = 8*0,4 = 3,2 (ok)

Exercicio 6 - Efeito de aumentar servidores (M/M/k)

Cenario: lambda = 18 req/min, mu = 10 req/min por servidor.

Com k=1: rho = 18/(1*10) = 1,8 -> sistema instavel (colapso) Com k=2: rho = 18/(2*10) = 0,9 -> sistema estavel, mas muito carregado Com k=3: rho = 18/(3*10) = 0,6 -> folga operacional maior Conclusao: Aumentar k reduz utilizacao por servidor e tende a diminuir fila e espera.

5) Quiz com Gabarito Comentado

  1. Quando um modelo analitico e preferivel?
    Quando o sistema e relativamente simples e as hipoteses matematicas sao plausiveis.
  2. Qual medida e mais robusta a outlier, media ou mediana?
    Mediana, porque outlier afeta muito menos sua posicao.
  3. O que significa p < 0,05?
    Ha evidencia estatistica para rejeitar H0 ao nivel de 5%.
  4. No M/M/1, o que ocorre quando lambda se aproxima de mu?
    rho aproxima de 1 e tempos de espera crescem fortemente.
  5. Explique M/D/3/10.
    Chegadas Markovianas, servico deterministico, 3 servidores, capacidade total 10 clientes.

1) O que é estrutura de dados?

Estrutura de dados é a forma como os dados são organizados na memória para permitir processamento eficiente. Em vez de guardar tudo como variáveis isoladas, o programa agrupa informações em formatos que facilitam buscar, alterar, remover e percorrer os valores.

  • Tipos primitivos: int, float, char, boolean.
  • Tipos construídos: vetores, matrizes, objetos, listas.
  • Objetivo: representar melhor problemas reais e melhorar desempenho.
Pergunta clássica: por que usar uma estrutura de dados? Resposta: para organizar os dados, otimizar o acesso e tornar a solução mais adequada ao problema.

2) Vetores

Vetor é uma estrutura homogênea unidimensional. Todos os elementos possuem o mesmo tipo e ficam em posições sequenciais na memória.

Características

  • Mesmo tipo para todos os elementos
  • Acesso por índice
  • Memória contígua
  • Tamanho normalmente definido na criação

Exemplo em Java

int[] valores = new int[5];
valores[0] = 10;
valores[1] = 20;

Vantagens

  • Implementação simples.
  • Acesso rápido a qualquer posição.
  • Menor gasto de memória do que listas com referência.

Desvantagens

  • Inserir no meio exige deslocar elementos.
  • Remover também pode exigir deslocamento.
  • Tamanho pode ser limitante.

3) Matrizes

Matriz é uma estrutura homogênea com duas ou mais dimensões. Ao contrário do vetor, exige mais de um índice para acessar cada posição.

int[][] matriz = new int[3][3];
matriz[0][0] = 1;
matriz[1][2] = 9;
  • Boa para tabelas, tabuleiros, mapas e imagens.
  • Em processamento de imagens, cada posição pode representar um pixel.
  • No caso RGB, a representação pode envolver três dimensões.

4) Alocação de memória e Java

Conceito Resumo
Alocação estática Espaço reservado com tamanho conhecido. Exemplo clássico: vetores e matrizes.
Alocação dinâmica Estrutura pode crescer ou diminuir durante a execução.
Referência Quando não é tipo primitivo, a variável aponta para um objeto.
JVM Gerencia a memória dos objetos Java e executa bytecode.
Lembre-se: em Java, quando você cria um objeto, a referência aponta para uma região de memória gerenciada pela JVM.

5) Estruturas estáticas x dinâmicas

Tipo Exemplo Comportamento
Estática Vetor / matriz Tamanho conhecido ou mais rígido
Dinâmica Lista encadeada Permite adicionar e remover elementos em tempo de execução

6) Listas encadeadas

Lista encadeada é uma estrutura dinâmica formada por nós. Cada nó guarda um dado e uma referência para o próximo elemento.

class Nodo {
    int dado;
    Nodo proximo;
}

Ideia central

  • Os elementos não precisam estar lado a lado na memória.
  • As referências ligam um nó ao outro.
  • Isso torna a inserção e a remoção mais flexíveis do que em vetores.

Simplesmente encadeada

Cada nó aponta apenas para o próximo.

Duplamente encadeada

Cada nó aponta para o próximo e para o anterior.

8) Vetor x lista encadeada

Critério Vetor Lista encadeada
Acesso por posição Rápido, geralmente $O(1)$ Precisa percorrer, geralmente $O(n)$
Inserção no meio Cara, precisa deslocar Mais flexível depois de achar a posição
Uso de memória Menor Maior por armazenar referências
Tamanho Mais rígido Mais dinâmico

9) Complexidade para decorar

  • Acesso por índice no vetor: $O(1)$
  • Busca linear: $O(n)$
  • Inserção no meio do vetor: $O(n)$
  • Remoção no meio do vetor: $O(n)$
  • Percorrer lista: $O(n)$
Mesmo quando a lista encadeada é melhor para inserir e remover, ainda pode haver custo para localizar a posição correta.

11) Mini revisão-relâmpago

  1. Vetor = homogêneo + índice + memória sequencial.
  2. Matriz = várias dimensões.
  3. Estrutura estática = tamanho mais rígido.
  4. Lista encadeada = nós + referências.
  5. Inserir/remover no meio do vetor tende a ser caro.
  6. Java usa JVM e referências para objetos.

12) O que não esquecer antes da prova

  • Não confunda acesso rápido com inserção rápida.
  • Não esqueça que lista encadeada precisa percorrer os nós para buscar.
  • Em Java, arrays e objetos são referências, e a JVM gerencia a memória.
  • Se pedirem vantagens/desvantagens, cite pelo menos duas de cada.
